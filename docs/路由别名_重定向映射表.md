# 路由别名/重定向映射表（历史兼容）

> 目的：本项目对“未交付/已改名”的入口不提供空壳页面，而是通过 redirect/alias 消除 404 与假闭环。  
> 为降低认知成本，本表列出当前仍保留的历史入口与其对应的主入口。

## 1) 映射表

| 历史/别名入口 | 当前主入口 | 说明 |
| --- | --- | --- |
| `/contacts` | `/crm/customers` | CRM 客户管理统一入口；历史入口保留重定向以兼容旧书签/旧导航。 |
| `/timelog` | `/time` | 计时追踪统一入口（避免多入口分裂）。 |
| `/cases/archive` | `/cases/archived` | 归档案件列表命名规范调整；保留旧入口重定向。 |
| `/settings` | `/profile` | 设置/个人中心统一入口（避免重复页面）。 |
| `/research` | `/tools` | “研究/工具”统一到工具箱入口，避免空壳或重复模块。 |

## 2) 维护规则（强制）

1. 如果新增/删除 redirect/alias，必须同步更新本表。
2. 变更后必须通过路由巡检：`pnpm -C lawclick-next audit:routes`（避免引用断链）。
3. 别名入口仅用于历史兼容，不应在新代码中作为“主入口”引用（新代码应使用“当前主入口”）。
