# 第一章：总体架构与设计哲学 (Architecture & Philosophy)

## 1.1 设计哲学：从“工具”到“生态”
LawClick V9.0 不再是一个单纯的 ERP 软件，而是一个**“可漂浮的、自由协作的律所生态系统 (Floating Legal Ecosystem)”**。

**核心变革：**
1.  **去中心化交互 (Floating UI)**：打破传统网页的框架限制。计时器、任务板、通讯窗不再死板固定，而是如乐高积木般可拆卸、悬浮、跨页面流转。利用浏览器的高级权限，实现“画中画”式的工作伴随。
2.  **深度赋能 (Deep Empowerment)**：
    *   **律师视角**：专注“生产”而非“记账”。提供智能文档审查、IM/邮件深度集成、RAG 知识检索，消除信息孤岛。
    *   **合伙人视角**：**摒弃传统的财务/发票逻辑**（交由专业财务软件），由“管钱”转向**“管体验 (CX)”**与**“管风险 (Risk)”**。
3.  **状态感知协作 (Context-Aware Collaboration)**：
    *   系统实时感知每个人的状态（忙碌/空闲/勿扰），并在协作前透传此信息。
    *   **“所见即所连”**：点击同事头像，不仅能聊天，更能看到他当前的公开工作上下文（如“正在撰写 XX 合同”），从而实现精准协作。

## 1.2 视觉与交互规范 (Visual Language)
*   **Rationalism UI (理性主义设计)**：沿袭 `LegalMind-Arbitration` 的高冷、专业、精密风格。
    *   **色彩**：以深邃蓝、岩石灰为主调，拒绝高饱和度的娱乐化配色。
    *   **排版**：严格的网格系统，衬线体（标题）与无衬线体（正文）的完美混排。
    *   **质感**：使用微磨砂玻璃 (Frosted Glass) 与细边框 (Hairline Border)，体现法律的严谨与通透。
*   **全中文本地化**：杜绝中英夹杂，所有术语必须符合大陆法系专业表述。

# 第二章：多维门户架构 (Multi-Dimensional Portals)

系统根据主体身份，动态重构整个界面逻辑，而非简单的权限开关。

## 2.1 律师工作台 (Lawyer Portal) - “深度工作舱”
*   **核心隐喻**：飞机的驾驶舱 (Cockpit)。
*   **功能焦点**：
    *   **分屏协作**：左侧沟通（IM/邮件），右侧作业（文书/案卷）。
    *   **智能文档**：不仅是存储，更支持在线批注、多人协同编辑、AI 审查。
    *   **自动化流转**：微信/邮件收到的证据，一键“存入案卷”，无需下载再上传。

## 2.2 合伙人/管理层门户 (Partner Portal) - “全景指挥塔”
*   **核心隐喻**：指挥塔 (Control Tower)。
*   **功能焦点**：
    *   **客户体验雷达 (CX Radar)**：实时监控客户满意度、案件响应速度、投诉预警。
    *   **风险控制中台**：穿透式利益冲突审查（基于图谱算法），时效熔断监控。
    *   **资源调度**：基于律师状态（忙/闲）的智能派单系统。

## 2.3 客户门户 (Client Portal) - “透明物流链”
*   **核心隐喻**：物流追踪 (Order Tracking)。
*   **功能焦点**：
    *   **极简进度条**：用通俗语言展示案件当前步骤（如“等待开庭”）。
    *   **自助服务**：在线预约会议、下载已归档文书、上传补充证据。

## 2.4 机构/外协门户 (Institution Portal)
*   **面向对象**：法院法官、检察官、鉴定机构。
*   **功能焦点**：
    *   **安全数据交换室 (Data Clean Room)**：用于与其进行合规的数据交换与文书送达。

# 第三章：核心业务模块重构

## 3.1 增强型协作生态 (Enhanced Collaboration Eco)
**“状态”是协作的基石。**
*   **智能状态灯 (Smart Status)**：
    *   系统根据计时器数据自动判断：
        *   🔴 **忙碌 (Do Not Disturb)**：正在进行“撰写代理词”等高专注任务（计时器运行中）。
        *   🟡 **会议中 (In Meeting)**：日程表显示当前有庭审或会议。
        *   🟢 **空闲 (Available)**：无计时任务，且鼠标活跃。
    *   **协作价值**：律师 A 想找 律师 B，看到 B 是红灯，系统自动建议“发送离线消息”或“预约后续时间”，避免打断心流。

## 3.2 深度文档工程 (Deep Document Engineering)
*   **版本重塑**：彻底取代“Windows 文件夹”。支持文件级、段落级的版本回溯。
*   **AI 审查 (The paralegal)**：
    *   上传合同 -> AI 自动高亮风险条款（基于律所知识库）。
    *   关联案例推荐：写到“不可抗力”时，侧边栏自动推送本所经办过的类似胜诉判例。

## 3.3 真正的“乐高式”组件 (Lego Components)
所有功能（计时、聊天、任务、日历）必须解耦为独立原子组件。
*   **可嵌入性**：聊天组件可以嵌入到“文档编辑页”右侧，也可以嵌入到“仪表盘”底部。
*   **可悬浮性**：计时器组件必须能脱离主窗口，以“画中画 (PiP)”或“系统级悬浮窗”形式存在，无论律师浏览什么网页，都能随时记录工时。
