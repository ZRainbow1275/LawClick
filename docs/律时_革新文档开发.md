我想开发一个基于flutter语言的，AI深度驱动嵌入的，名为「律时」的应用。该应用用于协助律师进行日常业务对接与跟踪、项目计时、律所内行政事务处理、客户交流；使得客户通过该软件可精准得知律师在自身项目中花费多少时间，处理何种业务。

该项目分为律师端和客户端两种身份和权限，同时对接律所的ERP系统进行使用。

律师端致力于实现：
1. 可使用语音或文字记录律师个人在某个项目上花费的时间
2. 基于可操作的日历表和时序图的叠加，可视化客户案件项目进展，并实时让客户得知进展
3. 基于2描述的可视化项目进展时间表，开发针对任何项目的某时间节点或任务节点类的留言板，以实现客户与律师的沟通
4. 类似mattermost留言区的项目制、小组制沟通功能，该功能中可实现律师与嵌入的AI进行对话
5. 进行AI驱动的合同审查与上传、项目追踪与管理（客户管理、立案管理、案件管理）、盖章文书管理等专业业务工作
  1. 合同审查与上传：AI驱动的合同文本OCR扫描识别、自动纠正、及时手动修改、加密时间戳与水印后上传到ERP系统
  2. 客户管理：AI驱动的客户信息填写，如姓名、单位名称、社会信用代码、是否上市
  3. 立案管理：AI驱动的立案信息填写，如案件名称、委托人、对方当事人、利益冲突审查情况、承办律师/团队、收费安排、审批状态等
  4. 案件管理：AI驱动的案件信息填写，如案件编号、收案日期、案件类型/业务领域、案由、委托代理合同等
  5. 盖章文书管理：需要律师盖章或者已经盖章的文书归档，涉及盖章的编号、申请人、文书类型、盖章类型、方式、份数等
6. 进行AI驱动的财务报销、事项审批、等业务行政业务工作。
  1. 特别的，在处理行政业务工作或者涉及到律所物料安排时，该系统应当可以借助提示，或者AI的回答，链接到律所的ERP系统，指出任何物品放置在哪个位置——这是该系统用于将律所信息化的重要步骤。



客户端致力于实现：

1. 通过上述对律师端欲意实现的功能的描述，及时地跟进并知情自身的项目进展
2. 通过上述对律师端欲意实现的功能的描述，及时地与律师进行沟通
3. 可与律所的AI进行对话
4. 及时进行盖章操作



从整体上来说，律师端致力于打造为一个律所数字化转型的，重点是由AI驱动和变得更加简单的重要前台，链接后端ERP。客户端致力于留住客户，让客户第一时间感受到律所的数字化面孔，和AI对话减少自身忧虑，同时项目制追踪可让客户更加放心自身项目的进展，不再停留于原本的“黑箱状态”之中。



因此，对于这两端的界面设计应当如下：

1. 通用的，首页将提供一个数字人形象接待。
2. 通用的，一个基于可操作的日历表和时序图的叠加的流程图
3. 其余界面请你根据上述描述的相关功能模块进行设计。
